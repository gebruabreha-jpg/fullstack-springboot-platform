tm = TaskManager()
This is where your class becomes a real working object.
1. TaskManager is just a blueprint
When you write:
class TaskManager:
    def __init__(self):
        self.tasks = []
        self.counter = 1

You are defining a class.
Think:

TaskManager = recipe

Not actual data yet.
No tasks exist.
No list exists yet.
Just a blueprint.
2. TaskManager() creates an object
This:
tm = TaskManager()
means:
👉 “Make a real object from this blueprint.”

Python:
allocates memory
runs __init__()
returns the object
So:
tm.tasks = []
tm.counter = 1
now actually exist.
Before object creation
Only class exists:
TaskManager (blueprint only)
After object creation
Real instance:
tm
 ├── tasks = []
 └── counter = 1
This is now alive in memory.
