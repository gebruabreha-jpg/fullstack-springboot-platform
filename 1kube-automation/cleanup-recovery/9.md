# Cleanup / Recovery Operations — Platform Engineering Guide

## Kubernetes Cleanup Patterns

### Post-Chaos Cleanup Steps

| Step | Command / Action | Purpose |
|------|------------------|---------|
| 1. Uncordon | `kubectlApi.uncordonNode(node)` | Make node schedulable again |
| 2. Cancel drain | `kubectlApi.cancelDrain(node)` | Cancel ongoing drain |
| 3. Restore interface | `ip link set <iface> up` | Bring network back up |
| 4. Restore tc | `tc qdisc del dev <iface> root netem` | Remove traffic control rules |
| 5. Delete RBAC | `kubectl delete -f rbac.yaml` | Clean up Litmus RBAC |
| 6. Delete experiment | `kubectl delete -f experiment.yaml` | Clean up ChaosExperiment |
| 7. Delete engine | `kubectl delete -f chaosengine.yaml` | Clean up ChaosEngine |
| 8. Litmus cleanup | `litmus.cleanupExperiment(id)` | Clean up Litmus resources |

### Cleanup Verification

- Verify all nodes are Ready and schedulable
- Verify no pods are in CrashLoopBackOff
- Verify no leftover ChaosEngine/ChaosResult resources
- Verify network interfaces are UP and tc rules are removed
- Verify all PVCs/PVs are in expected state (Bound or Released)
